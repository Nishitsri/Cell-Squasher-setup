5     Short Introduction to sampling the ADS1115 using C

 

Here a short example of using C to initiate and read a single conversion from the ADS1115. For the sake of simplicity there is no error checking as I feel it makes the code more readable. The code is based around the <linux/i2c-dev.h> header which gives us the function calls needed to communicate with an I2C device. This page provides a more general guide on how to use this library (including error checking):

 

https://www.kernel.org/doc/Documentation/i2c/dev-interface

 

Some other guides on using I2C using C (not necessary to read these):

 

http://elinux.org/Interfacing_with_I2C_Devices

http://blog.chrysocome.net/2013/03/programming-i2c.html

http://blog.davidegrayson.com/2012/06/using-ic-on-raspberry-pi.html

 

The important thing to note is that the slave address byte mentioned in section 2.2.2 is transmitted for us, we dont need to specify this byte. We choose whether to read or write from the device using either the read() or write() function calls.

 

Code needed to take a single sample:

 

#include <stdio.h>
#include <fcntl.h>     // open
#include <inttypes.h>  // uint8_t,uint16_t
#include <linux/i2c-dev.h> // I2C bus definitions

int main() {
    
  int ADS_address = 0x48;
  int I2CFile;
  uint8_t writeBuf[3];
  uint8_t readBuf[2];
  int16_t reading;
  
  I2CFile = open("/dev/i2c-1", O_RDWR);
  
  ioctl(I2CFile, I2C_SLAVE, ADS_address);
      
  writeBuf[0] = 1;
  writeBuf[1] = 195;
  writeBuf[2] = 3;
  
  readBuf[0]= 0;
  readBuf[1]= 0;
      
  write(I2CFile, writeBuf, 3);

  while ((readBuf[0] & 0x80) == 0)
  {
      read(I2CFile, readBuf, 2);
  }
    
  writeBuf[0] = 0;
  write(I2CFile, writeBuf, 1);
  
  read(I2CFile, readBuf, 2);

  reading = readBuf[0] << 8 | readBuf[1];
  printf("Voltage Reading %f (V) \n", (float)reading*4.096/32767.0);
        
  close(I2CFile);
  
  return 0;

}

 

#include <linux/i2c-dev.h> // I2C bus definitions

 

Including this header file gives us the function calls we need to read and write to our I2C device

 

  uint8_t writeBuf[3];
  uint8_t readBuf[2];
  int16_t reading;

 

Here we create two buffers containing unsigned 8 bit integers, the first stores the three bytes that will be written to the device, the second will contain the two bytes read from the device. Remember that when writing to the device we must send an extra byte that sets the value of the pointer register, this is why this buffer is three rather than two bytes long. Then we create an unsigned 16 bit integer called reading to store the result of the conversion. We are performing a single ended conversion in this example, if you were going to use a differential conversion reading would need to be signed.

 

  I2CFile = open("/dev/i2c-1", O_RDWR);

 

Here we open the I2C device.

 

  ioctl(I2CFile, I2C_SLAVE, ADS_address);

 

This specifies the address of the slave we want to communicate with.

 

  writeBuf[0] = 1;
  writeBuf[1] = 195;
  writeBuf[2] = 3;
  
  readBuf[0]= 0;
  readBuf[1]= 0;

 

The first byte in the write buffer is set to 1, this sets the pointer register so that the next two bytes are written to the config register. The second and third bytes written to the register follow directly from section 2.2.1. We also initialize readBuf to 0.

 

  write(I2CFile, writeBuf, 3);

 

Here we write writeBuf to the device. The 3 specifier in the function is the number of bytes to be transmitted.

 

  while ((readBuf[0] & 0x80) == 0)
  {
      read(I2CFile, readBuf, 2);
  }

 

This section of code waits until the conversion is complete. While a conversion is taking place bit 15 in the config register is set to 1, and once the conversion is finished the bit is set to 0. Here we continually read the contents of the config register storing the result in readBuf. We then check if bit 15 is equal to 0, and if so break out of the while loop.

 

  writeBuf[0] = 0;
  write(I2CFile, writeBuf, 1);

 

Once the conversion is complete we need to change the pointer register so that we can read from the conversion register. This requires the pointer register to be set to 0.

 

  read(I2CFile, readBuf, 2);

 

Here we read the value of the conversion register and place the result in readBuf.

 

  reading = readBuf[0] << 8 | readBuf[1];
  printf("Voltage Reading %f (V) \n", (float)reading*4.096/32767.0);

 

Finally we take the two bytes stored in readBuf and combine them to form the 16 bit result of the conversion which is stored in the variable reading. We then print the result of the conversion to the terminal with the sixteen bit value is first converted to the value of the reading in mV. 4.096 is the FS range set by the programmable gain amplifier, 32767 is 215-1 (15 bit reading as this is a single ended conversion).

 

To run the above program navigate to the directory in which it is saved and then compile the code (here my program is called sampledata.c):

 

gcc o sampledata sampledata.c

 

Then run the program by typing:

 

sudo ./sampledata

 

When I was looking around for examples on how to communicate with this device I found this forum post, which the code I gave above is loosely based on:

 

http://www.raspberrypi.org/phpBB3/viewtopic.php?t=36250&p=305270

 

The code in this post carries out the same function as the code I gave but includes error checking. However the poster was asking for help as to why their code wasnt functioning correctly. I think the issues with the code were:

 

buf[1] = 0xc3;

 

Should be:

 

buf[1] = 0xc2;

 

0xc3 starts continuous conversion while what we want is just a single conversion.

 

  } while (buf[0] & 0x80 == 0);

 

Should be:

 

  } while ((buf[0] & 0x80) == 0);

 

Otherwise the while statement always returns true.
